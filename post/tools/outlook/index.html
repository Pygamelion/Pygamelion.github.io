WARNING

尝试使用windows自带的Outlook软件来使用gmail邮箱和新注册的outlook邮箱，今天和0x80190001斗争良久，虽然还是不太清楚发生了什么，但是一顿乱搞之后搞定了，将情况记录如下以备日后参考：

登录报错，原因不详

虽然我在第二台电脑上直接登录就OK… 我想可能是微软服务验证同步了，好像不能两台电脑上同时登录 outlook 邮箱?

可能的配置方案

我猜测这是一个主要步骤？依次执行netsh winhttp reset proxy, netsh winhttp import proxy source=ie, 好像是把 IE / 系统代理同步到 WinHTTP，保证 Outlook 等程序也能用？总之netsh winhttp show proxy的结果配置如下：

当前的 WinHTTP 代理服务器设置:

    代理服务器:  127.0.0.1:7897
    绕过列表     :  localhost;127.*;192.168.*;10.*;172.16.*;172.17.*;172.18.*;172.19.*;172.20.*;172.21.*;
    172.22.*;172.23.*;172.24.*;172.25.*;172.26.*;172.27.*;172.28.*;172.29.*;172.30.*;172.31.*;<local>

绕过列表好像不一定需要配置…..

不太清楚发生了什么

登录google邮箱时开代理，从浏览器确认; 登录outlook邮箱时关掉代理，直接登录就能连接进入。

之后再开关代理就不会报错了？好像是微软在做账户验证(￣ ▽ ￣)”
